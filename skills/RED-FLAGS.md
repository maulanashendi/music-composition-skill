# Red flags ‚Äî common excuses vs. reality

Shared across the 3 MDLC skills ‚Äî `jazz-composing`, `plan-verifying`,
`rendering-audition` ‚Äî plus the legacy `abc-notation` path. When you catch
yourself thinking one of the excuses below, stop and check the reality
column before delivering.

| Excuse (what you might tell yourself) | Reality (what's actually true) | Where to check |
|---|---|---|
| "This section needs every device to prove it's sophisticated." | Stacking every technique is how a section turns into noise. The arc decides what's used; most sections need 2-3 moves, not all of them. | skills/jazz-composing/SKILL.md "the one rule"; skills/jazz-composing/references/arrangement.md interaction-map rule |
| "The chorus should basically repeat the verse ‚Äî it's a loop, that's fine." | A dramatic arc that never changes isn't an arc. Verse and chorus (or any repeated section) need audible development ‚Äî register, density, harmony, or interaction shift ‚Äî not identical restatement. | skills/jazz-composing/references/loop-development.md, arrangement.md |
| "The drum grid and the ABC don't line up bar-for-bar, but it's close enough." (legacy ABC path) | Close enough drifts into audible desync as the piece goes on. Bar counts between the drum step-grid and the ABC must match exactly, section by section. On the current default path this concern moves to `plan.json`/`pyengine`, which enforces bar-matching structurally. | skills/abc-notation/references/drums-and-abc.md, common-failures.md (legacy ABC path only) |
| "Chord symbols in the ABC will just be read as chords, not played as notes." (legacy ABC path) | Some converters (including this package's own `abc_to_midi.py`) will sound a bare chord-symbol string as literal notes unless it's stripped. Always verify the converter's actual output, not just the ABC's visual intent. | skills/abc-notation/scripts/abc_to_midi.py (chord-symbol stripping) |
| "`validate_abc.py` passed with 0 errors, so the piece is fine." (legacy ABC path) | The validator checks ABC structure only ‚Äî it does not know whether `abc_to_midi.py` will actually produce notes for every voice, or whether the merged MIDI's tempo/time-signature tag matches the ABC header. A file can pass validation and still render silently wrong. Always inspect the actual MIDI output (track count, note count per track, tempo/time-signature) as a separate check. On the plan.json default path, the equivalent trap is "`pyengine validate` passed, so the render is fine" ‚Äî validate only checks structural/craft warnings, not whether the actual rendered audio sounds right; ear-check is still required. | tests/results/2026-07-13-brief-01-02-armA-vs-armB.md ‚Äî this exact failure was found twice by independent eval runs before being fixed |
| "The multi-voice ABC looks right ‚Äî voices are declared, that's enough." (legacy ABC path) | This package's `abc_to_midi.py` only captures lines that literally start with `[V:id]`; a continuation line without the marker is silently dropped (0 notes for that voice) even though the file still validates. Repeat `[V:id]` on every content line for that voice. | skills/abc-notation/references/abc-syntax.md ¬ß14, common-failures.md |
| "The rubric score is high, so the piece must sound good." | The rubric checklist scores craft and theory-consistency from the plan/notation ‚Äî it cannot hear timbre, groove feel, or mix. A high score is a floor (nothing is broken or thin), not a ceiling (nobody has confirmed it's enjoyable). Ear-check is still required before calling a piece finished. | skills/rendering-audition/references/rubric-checklist.md; skills/rendering-audition/references/scorecard-template.md; skills/rendering-audition/references/audition-protocol.md |
| "Motif ini 4-not, arpeggio-nya outside, chord-nya borrowed ‚Äî sesuai yang kutulis." | Label teori di prosa sering tak cocok notasi aktual: motif "4-not" yang nyatanya 3 pitch (rest bukan pitch), arpeggio chord-tone yang dilabeli "outside", chord diatonik yang disebut "borrowed", klaim top-note yang beda dari voicing tertulis. Jangan percaya label sebelum dicek terhadap fakta notasi yang sebenarnya berbunyi. | skills/abc-notation/scripts/notation_facts.py (jalankan pada ABC-nya; gate wajib di fase Ideation ‚Äî legacy ABC path. Di jalur default `plan.json`, kesetaraan gate ini adalah `pyengine validate` + audit rubrik/telinga di `rendering-audition`) |
| "First draft-nya sudah solid ‚Äî tidak ada yang perlu direvisi." | Run tanpa satu pun revisi tercatat berarti temuan rubrik/blind/cliche tidak pernah benar-benar dijawab ‚Äî "first draft dianggap final" adalah pola gagal, bukan bukti kualitas. Sebelum run disebut selesai wajib ada bagian "Bukti revisi": ‚â•1 pasangan before/after untuk 2 masalah terbesar, terhubung ke temuan nyata. | skills/rendering-audition/SKILL.md (fase Review); skills/rendering-audition/references/scorecard-template.md bagian "Bukti revisi" |
| "Device ini memang disengaja ‚Äî sudah kubilang begitu." | Justifikasi tanpa mekanisme audible yang konkret bukan justifikasi ‚Äî itu pembelaan. Temuan cliche-register hanya boleh dijawab dengan revisi ATAU justifikasi audible spesifik (mekanisme yang bisa didengar kenapa device dipertahankan); "ini disengaja" polos tidak diterima. | skills/jazz-composing/references/cliche-register.md; skills/rendering-audition/references/scorecard-template.md bagian cliche |
